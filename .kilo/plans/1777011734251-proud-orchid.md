# Implementation Plan: Android Compatibility & TelegramClient Enhancements

## Overview
Update the TelegramClient and related Android components with modern compatibility handling, focusing on:
1. Android version-aware utilities (ApiCompat)
2. Storage permission handling for Android 13+
3. Background location for Android 10+
4. Foreground service requirements (Android 8+)
5. WiFi scanning on Android 10+
6. CameraX for modern camera handling
7. Notification permission for Android 13+

## Completed Changes ✓

### 1. ApiCompat Module
- **File**: `app/src/main/java/com/example/agent/modules/ApiCompat.kt`
- **Purpose**: Centralized Android version checks
- **Features**:
  - `isOreo`, `isQ`, `isR`, `isS`, `isTiramisu`, `isUpsideDownCake` properties
  - `storagePermissions()` returns correct storage permissions by Android version
  - `hasPermission()` / `hasAll()` permission check helpers

### 2. TelegramClient Upgrades
- **File**: `app/src/main/java/com/example/agent/TelegramClient.kt`
- **Changes**:
  - Added `sendLocation()` for GPS pin sharing
  - File size validation (50MB) with user feedback
  - Audio duration parameter support
  - Improved multipart upload with chunked streaming
  - Kotlin built-in `.chunked()` instead of custom helper
  - Better HTTP handling (UTF-8, cleaner error handling)
  - Comprehensive documentation

### 3. Version Check Consolidation
- **Updated Files**:
  - `MainActivity.kt` - All Build.VERSION checks → ApiCompat properties
  - `ShellCapability.kt` - Storage directory selection
  - `LocationCapability.kt` - Background location handling
  - `FileCapability.kt` - Storage permission check
  - `BootReceiver.kt` - Foreground service start

## Remaining Implementation Required

### Phase 1: Storage Permissions (Android 13+)
**Priority**: HIGH

**Changes Needed**:
1. **AndroidManifest.xml**
   - Remove: `READ_EXTERNAL_STORAGE`, `WRITE_EXTERNAL_STORAGE` (deprecated)
   - Add: `READ_MEDIA_IMAGES`, `READ_MEDIA_VIDEO`, `READ_MEDIA_AUDIO`
   - Keep: `READ_EXTERNAL_STORAGE` for Android < 13 (backward compatibility)

2. **MainActivity.kt** - Update `requiredPermissions` list
   - Use `ApiCompat.storagePermissions()` instead of hardcoded array

3. **FileCapability.kt** - Update permission checking
   - Replace manual check with `ApiCompat.hasPermission()` calls

### Phase 2: Background Location (Android 10+)
**Priority**: MEDIUM

**Changes Needed**:
1. **AndroidManifest.xml**
   - Add: `<uses-permission android:name="android.permission.ACCESS_BACKGROUND_LOCATION" />`

2. **MainActivity.kt** - Already adds background location permission for Q+
   - Currently adds `ACCESS_BACKGROUND_LOCATION` when `ApiCompat.isQ`

3. **LocationCapability.kt**
   - Background location requires separate user approval in Settings
   - Consider adding user guidance message

### Phase 3: Foreground Service (Android 8+)
**Priority**: HIGH

**Changes Needed**:
1. **AndroidManifest.xml**
   - Add `android:foregroundServiceType` to service declaration
   - Include types: `location`, `dataSync`, `camera`, `microphone`

2. **AgentService.kt**
   - Verify: Currently calls `startForeground()` with notification
   - Ensure called within 5 seconds of `startForegroundService()`

3. **Notification**
   - Must have proper notification channel (API 26+)
   - Should NOT use `IMPORTANCE_MIN` (too hidden)

### Phase 4: WiFi Scanning (Android 10+)
**Priority**: MEDIUM

**Changes Needed**:
1. **NetworkCapability.kt**
   - Update `getWifiScanResults()` function
   - Must register BroadcastReceiver for `SCAN_RESULTS_AVAILABLE_ACTION`
   - Request `ACCESS_FINE_LOCATION`
   - Handle throttling (max 4 per 2 minutes)

### Phase 5: CameraX Integration
**Priority**: LOW (future enhancement)

**Changes Needed**:
1. **build.gradle.kts (app)**
   - Add CameraX dependencies

2. **CameraCapability.kt**
   - Replace Camera2 API with CameraX

### Phase 6: Notification Permission (Android 13+)
**Priority**: MEDIUM

**Changes Needed**:
1. **AndroidManifest.xml**
   - Add: `POST_NOTIFICATIONS` permission

2. **MainActivity.kt**
   - Add notification permission for Tiramisu+

3. **AgentService.kt**
   - Ensure notification importance not `IMPORTANCE_MIN`
